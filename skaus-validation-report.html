What You're Building

Core Innovation

SKAUS is a privacy-preserving payment protocol on Solana. Senders deposit SOL/USDC into a shared stealth pool; recipients withdraw using Groth16 ZK proofs — without revealing their identity or wallet. On top of this, a human-readable pay-link layer (e.g. skaus.me/alice/coffee-tips) provides the Stripe-like UX layer for creators, freelancers, and businesses.

Key Differentiators vs. Alternatives

  • Gasless relayer: Recipients never pay gas — relayer submits withdrawal and deducts a small fee from the amount.
  • PIN-based key management: No cloud key storage; keys derived on-demand from PIN + wallet signature via HKDF. No separate hardware wallet required.
  • Human-readable pay links: Username-based URLs with full link analytics (views, payment counts, revenue). Competes with Helio/TipLink on UX, not just crypto.
  • Anonymity set uniformity: Fixed deposit tiers (power-of-10) ensure every note in the pool is indistinguishable — stronger privacy guarantees than variable-amount schemes.

Crypto Necessity Check

✓ Crypto is necessary — the blockchain is load-bearing

Without Solana: You'd need a custodian to hold funds and a central server to route payments — that's just Venmo with extra steps, and the privacy claim collapses the moment the custodian is subpoenaed. The Groth16 ZK proof, nullifier-based double-spend prevention, and on-chain merkle tree are all doing real work. Removing the blockchain breaks the product entirely.

Integration vs. Build Assessment

Custom Build — Correct Call

SKAUS is building the protocol layer itself (stealth pool program, ZK verifier, name registry). This is not a product that can be assembled from existing Solana protocols — the core value is the novel on-chain logic. Integrate-first would have meant no privacy guarantees and no novel moat. The custom build decision is validated.

Where integration makes sense going forward

  • Token infrastructure: Use SPL Token / Token-2022 natively — no need to build token handling.
  • Fiat off-ramp: Integrate Helio or MoonPay for merchants who want auto-fiat settlement.
  • Indexing: Consider Helius webhooks or Triton to replace the custom deposit indexer polling loop.
  • Analytics: Use existing dashboard SDKs for pay link analytics rather than building from scratch.

Next Steps

Add compliance hooks before mainnet launch

Implement selective disclosure: an opt-in mode that generates an audit trail (HMAC-signed spend log) that only the user can share with regulators. This removes the biggest legal liability and enables enterprise adoption. See Solana's Confidential Balances framework for precedent.

Differentiate from GhostWare via pay link UX — ship it fast

Your human-readable pay links, QR codes, link analytics, and gasless withdrawal are already built. GhostWare does not have this. Publish a public demo and tweet about it. This is your current moat and it won't last long.

Target B2B for initial revenue: on-chain payroll + DeFi protocols

Reach out to Streamflow, Zebec (GhostWare's pilot partner — steal the deal), and any DeFi protocol that pays contributors on-chain. These buyers understand privacy, have budget, and make purchase decisions fast.

Apply for Solana Foundation Privacy Hack grants

The Foundation is actively funding privacy tooling. SKAUS is exactly what they want to fund. Apply immediately. This also serves as a credibility signal for enterprise buyers.

Benchmark ZK proof generation on low-end devices before mainnet

Run proof gen on a budget Android device (Snapdragon 662 or equivalent). If it takes >5 seconds, optimize the WASM bundle or introduce a relayed proving fallback (user sends encrypted note, relayer generates proof). Decide before mainnet not after.

Define a monetization path beyond relay fees

Relay BPS fees are correct for bootstrapping but thin at scale. Design a "Business tier" for pay links: custom branding, webhook notifications, team access, CSV export. This creates recurring SaaS revenue on top of the protocol layer.
